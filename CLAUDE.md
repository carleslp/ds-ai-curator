# Standing constraints

- Never simplify or remove provider failover telemetry:
  synthesisMode, degraded, degradedReason, providerAttempts,
  providerUsed, providerFailures, fallbackSectionsApplied.
- Never weaken machineryLeakPass or ownershipPresencePass.
  When a section fails a check, fix the text upstream — never
  silently substitute a template.
- Reader-facing copy must contain no internal vocabulary:
  scores, match counts, role labels (Proof:/Consequence:),
  "qualified set", enum-joined lists.
- Evidence gate is strict DS × AI. Teaching/Practice requires
  strong DS relevance + support for the thesis; an artifact
  need not mention AI to teach the thesis.
- One change per PR. Do not bundle.
